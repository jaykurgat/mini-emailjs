Mini-EmailJS Docs

Mini-EmailJS is a self-hosted transactional email platform for website contact forms. Connect any site's form to your own email inbox — using Resend or your own Gmail — with a single fetch() call.

Each website you connect is a project, with its own API endpoint, API key, email provider, submission log, and rate limits.

Self-hosted: You own the infrastructure, the data, and the email-sending credentials. No third-party dashboard. No monthly submission limits beyond your email provider's free tier.

Quick Start

1. Create a project

In the dashboard, click New Project. Give it a name, enter the email address where you want to receive notifications, choose a provider (Resend or Gmail SMTP), and click Create.

You'll land on the Integration Hub with your API endpoint and key ready to copy.

2. Add the snippet to your site

Use the Form Builder to generate a styled, ready-to-paste form, or wire up your existing form manually:

const ENDPOINT = "https://your-deployment.vercel.app/api/send/your-project-id";
const API_KEY  = "mek_your_api_key_here";

document.getElementById('contact-form').addEventListener('submit', async (e) => {
  e.preventDefault();
  const data = Object.fromEntries(new FormData(e.target).entries());
  data.apiKey = API_KEY;

  const res = await fetch(ENDPOINT, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data),
  });
  const result = await res.json();
  if (result.ok) alert('Message sent!');
});

3. Test it

Fill out your form and submit. The notification email should arrive within seconds. Check Overview in the dashboard to see the submission logged.

How It Works

When your form submits, the browser sends a POST request to your deployment's /api/send/:projectId endpoint. The serverless function:

  1. Looks up the project in Supabase by projectId
  2. Validates the apiKey against the stored key
  3. Checks the request origin against the project's allowed origins list
  4. Checks the honeypot field — silently drops bot submissions
  5. Enforces the per-IP rate limit
  6. Renders the email subject from the project's subject template
  7. Sends the notification email via the project's configured provider (Resend or Gmail SMTP)
  8. Optionally sends an auto-reply confirmation to the submitter
  9. Logs the submission (status, payload, IP, timestamp) to Supabase

The entire flow is serverless — no persistent process, no server to maintain. Each form submission triggers a fresh function invocation on Vercel.

POST /api/send/:projectId

The only public-facing endpoint. Called directly from your website's form submission handler.

POST https://your-deployment.vercel.app/api/send/{projectId}
Content-Type: application/json

projectId is the URL slug you chose when creating the project (visible in the dashboard and Integration Hub).

Request Body

Send a JSON object. The only required field is apiKey — all other fields are your form's own data and will be included in the notification email exactly as submitted.

FieldTypeRequiredDescription
apiKeystringYesYour project's API key. Found in Integration Hub.
from_namestringNoSubmitter's name. Used in subject template and auto-reply.
emailstringNoSubmitter's email. Used as Reply-To and for auto-reply delivery.
_gotchastringNoHoneypot field. Must be empty — bots often fill it in.
Any other fieldanyNoIncluded in the notification email. Use any field names your form needs.
 
{
  "apiKey":    "mek_your_key_here",
  "from_name": "Jane Smith",
  "email":     "jane@example.com",
  "phone":     "+1 555 000 0000",
  "message":   "Hi, I'd like to book a diagnostic session."
}

Responses

StatusBodyMeaning
200{"ok": true}Submission accepted and email sent.
401{"ok": false, "error": "Invalid API key."}The apiKey field is missing or wrong.
403{"ok": false, "error": "Origin not allowed."}The request origin is not in the project's allowed origins list.
404{"ok": false, "error": "Unknown or inactive project."}The projectId in the URL doesn't exist or is paused.
429{"ok": false, "error": "Too many submissions."}This IP has exceeded the project's rate limit.
500{"ok": false, "error": "Internal server error."}Something went wrong server-side. Check Vercel function logs.
Note: A 200 response with {"ok": true} is also returned when a honeypot submission is silently dropped — this prevents bots from knowing their submission was blocked.

Error Codes

All error responses follow the same shape: {"ok": false, "error": "human-readable message"}. Your form's submit handler should check result.ok and display result.error to the user if it's false.

JavaScript / Fetch

const ENDPOINT = "https://your-deployment.vercel.app/api/send/your-project-id";
const API_KEY  = "mek_your_api_key_here";

async function submitForm(formData) {
  const data = Object.fromEntries(formData.entries());
  data.apiKey = API_KEY;

  try {
    const res = await fetch(ENDPOINT, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    });
    const result = await res.json();

    if (result.ok) {
      // Show success message to user
      return { success: true };
    } else {
      // Show error message to user
      return { success: false, error: result.error };
    }
  } catch (err) {
    // Network error
    return { success: false, error: 'Network error. Please try again.' };
  }
}

Python

import requests

ENDPOINT = "https://your-deployment.vercel.app/api/send/your-project-id"
API_KEY  = "mek_your_key_here"

response = requests.post(ENDPOINT, json={
    "apiKey":    API_KEY,
    "from_name": "Jane Smith",
    "email":     "jane@example.com",
    "message":   "Hello from Python!"
})

result = response.json()
if result.get("ok"):
    print("Email sent successfully!")
else:
    print(f"Error: {result.get('error')}")

Plain HTML Form

If you prefer not to use JavaScript, you can wire up a plain HTML form. Note that this approach doesn't give you a seamless single-page experience — the form will submit and the page will reload.

<form id="contact-form">
  <input type="text"  name="from_name" placeholder="Your name"    required/>
  <input type="email" name="email"     placeholder="Your email"   required/>
  <textarea           name="message"   placeholder="Your message" required></textarea>

  <!-- Honeypot: hidden from users, bots often fill it in -->
  <input type="text" name="_gotcha"
    style="position:absolute;left:-9999px" tabindex="-1" autocomplete="off"/>

  <button type="submit">Send</button>
  <div id="form-status"></div>
</form>

<script>
document.getElementById('contact-form').addEventListener('submit', async (e) => {
  e.preventDefault();
  const status = document.getElementById('form-status');
  const data = Object.fromEntries(new FormData(e.target).entries());
  data.apiKey = "mek_your_key_here";

  status.textContent = 'Sending…';
  const res    = await fetch("https://your-deployment.vercel.app/api/send/your-project-id", {
    method: 'POST', headers: {'Content-Type':'application/json'}, body: JSON.stringify(data),
  });
  const result = await res.json();
  status.textContent = result.ok ? 'Message sent!' : 'Error: ' + result.error;
});
</script>

Spam Protection (Honeypot)

Every project has a honeypot field configured (default: _gotcha). Add a hidden input with this name to your form — real users won't see or fill it, but many bots will.

<!-- Add this to your form -->
<input type="text" name="_gotcha"
  style="position:absolute;left:-9999px"
  tabindex="-1" autocomplete="off"/>

When the honeypot field has a value, the submission is silently dropped — a 200 {"ok": true} is returned to the bot so it doesn't retry.

Tip: The generated forms from the Form Builder include the honeypot field automatically.

Rate Limiting

Each project has a configurable rate limit — the maximum number of submissions allowed per IP address per hour. The default is 20 per hour, which is generous for a typical contact form.

When the limit is exceeded, the endpoint returns 429 Too Many Requests:

{"ok": false, "error": "Too many submissions. Please try again later."}

Adjust the rate limit in Project Settings.

Auto-Reply

When enabled, the platform automatically sends a confirmation email to the form submitter after their submission is processed. Requires an email field to be present in the submission.

Configure auto-reply in the Automation section of your project. The subject and body support {{field_name}} placeholders:

Subject: Thanks for reaching out, {{from_name}}!

Body:
Hi {{from_name}},

Thanks for getting in touch. We've received your message
and will get back to you within one business day.

Best,
The Team
Note: If the auto-reply fails (e.g. invalid email address), the main notification email is still sent and the submission is still logged. Auto-reply failures are non-blocking.

Allowed Origins

You can restrict which websites are allowed to call your project's endpoint by setting the Allowed Origins field in Project Settings.

Enter a comma-separated list of origins without trailing slashes:

https://example.com, https://www.example.com, https://staging.example.com

Leave blank to allow requests from any origin (useful during testing, but tighten before going to production).

The comparison is case-sensitive and exact — https://example.com and https://example.com/ are treated as the same (trailing slashes are automatically stripped).

API Keys

Each project has a unique API key in the format mek_ followed by 48 random hex characters. This key is required in every submission request as the apiKey field.

Security note: Your API key will be visible in your website's client-side JavaScript — this is intentional and by design (the same model as EmailJS, Stripe's publishable key, etc.). The key only grants the ability to submit a form to that specific project. It cannot access your dashboard, view submissions, or modify project settings. Restrict the allowed origins to limit which sites can use it.

Rotating API Keys

If you believe your API key has been compromised, rotate it immediately from the Integration Hub by clicking Regenerate Key. The old key stops working instantly.

After rotating, update the key in any website that was using the old one — otherwise their forms will start returning 401 Invalid API key errors.

All submissions made with the old key before rotation remain in the submission log.
